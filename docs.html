TUI Usage

# Interactive TUI
agem

# Open with an initial prompt
agem "explain the architecture of this codebase"

# Non-interactive — run one prompt and exit
agem --message "add JSDoc to every exported function in src/utils/helpers.ts"

# Resume last session
agem --resume

# Resume a specific session
agem --resume <session-id>

# List saved sessions
agem --list-sessions

Gateway Daemon

The gateway is a persistent background process that keeps the agent alive between TUI sessions and exposes a WebSocket + HTTP interface for external clients.

# Start the daemon
agem gateway start

# Check status
agem gateway status

# Stop the daemon
agem gateway stop

# Custom ports
agem gateway start --ws-port 7878 --http-port 7879 --host 127.0.0.1

When the daemon is running, the TUI connects automatically (shown as ⬡ gw in the header). Falls back to direct mode if no daemon is found.

Hot Reload (SIGHUP)

Send SIGHUP to reload config and reconnect channels without downtime:

kill -HUP $(cat ~/.agem/gateway.pid)

Hot reload re-reads config, diffs channel credentials, reconnects changed channels, and reinitialises the job scheduler — leaving active sessions untouched.

WebSocket API

Connect to ws://127.0.0.1:7878 and send JSON messages:

// Create a session
{ "kind": "session.create", "cwd": "/path/to/project" }

// Send a message
{ "kind": "chat", "sessionId": "<id>", "text": "your prompt" }

// List sessions
{ "kind": "session.list" }

// Interrupt a running turn
{ "kind": "interrupt", "sessionId": "<id>" }

Inbound kinds: ping, chat, session.create, session.resume, session.list, session.close, interrupt

Outbound kinds: pong, session.created, stream.delta, stream.done, message.assistant, tool.call, tool.result, approval.request, error, status

Job Scheduler

Create one-shot, interval, or cron jobs via the REST API:

# Heartbeat every 5 minutes
curl -X POST http://127.0.0.1:7879/jobs \
  -H "Content-Type: application/json" \
  -d '{"name":"heartbeat","schedule":{"every":300000},"payload":{"kind":"heartbeat"}}'

# Run a shell command on a cron schedule
curl -X POST http://127.0.0.1:7879/jobs \
  -d '{"schedule":{"cron":"0 9 * * 1-5"},"payload":{"kind":"shell","command":"git pull"}}'

# One-shot agent message
curl -X POST http://127.0.0.1:7879/jobs \
  -d '{"schedule":{"at":"2026-04-02T09:00:00Z"},"payload":{"kind":"agent","sessionId":"<id>","message":"run the test suite"}}'

Channel Integrations

The gateway bridges external chat platforms to the agent. Each platform gets its own isolated agent session per conversation context.

Discord

pnpm add discord.js
export AGEM_DISCORD_TOKEN=your-bot-token
agem gateway start

Responds when @mentioned in guild channels or in DMs. Session keys: discord:guild:<guildId>-<channelId>, discord:dm:<userId>.

Telegram

pnpm add grammy
export AGEM_TELEGRAM_BOT_TOKEN=your-bot-token
agem gateway start

Responds to all private messages and to group messages where @mentioned. Session keys: telegram:chat:<chatId>.

WhatsApp

pnpm add @whiskeysockets/baileys
agem gateway start  # QR code appears; scan with WhatsApp

Auth credentials saved to ~/.agem/wa-auth/ — scan once, reconnects automatically. AGEM_WA_GROUPS=1 responds in all group messages.

Matrix

pnpm add matrix-js-sdk
export AGEM_MATRIX_HOMESERVER=https://matrix.org
export AGEM_MATRIX_USER_ID=@yourbotname:matrix.org
export AGEM_MATRIX_TOKEN=<access-token>
agem gateway start

Invite the bot to any room and it joins automatically. AGEM_MATRIX_MENTION_ONLY=0 responds in all rooms regardless of mention.

Execution Modes

Every tool call is classified before execution:

ModeRead-only toolsWrite toolsShell / tasks
auto-approve✅ Allow✅ Allow✅ Allow
normal✅ Allow⚠️ Prompt⚠️ Prompt
read-only✅ Allow❌ Deny❌ Deny
plan✅ Allow❌ Deny❌ Deny

⚠️ Note: auto-approve allows the agent to write files and run shell commands without confirmation. Use normal mode when working in unfamiliar codebases.

MCP Integration

Add MCP servers to .agem/mcp.json (project) or ~/.agem/mcp.json (global):

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
    },
    "remote-api": {
      "url": "http://localhost:3001/sse",
      "headers": { "Authorization": "Bearer <token>" }
    }
  }
}

Use /mcp in the TUI to list connected servers and their tools. MCP health is shown as colored dots in the header.

Project Instructions

Create .agem/instructions.md in your project root to give the agent persistent context — coding conventions, architecture notes, or task-specific guidance. Loaded automatically on startup and injected into every conversation turn.

## Conventions
- Use TypeScript strict mode
- Prefer functional components in React
- All exports must have JSDoc comments

## Architecture
The app follows a layered architecture:
- src/api/     → REST endpoint handlers
- src/domain/  → Business logic, no framework deps
- src/infra/   → Database, external services
