Changelog

All notable changes to AgentEmerald, in reverse chronological order.

Unreleased

Unreleased

2026-04-05

Gateway auto-start

  • src/gateway/autoStart.ts — ensureGateway() automatically starts the daemon when the TUI launches. Probes /health, spawns daemon if needed, polls until ready.
  • TUI startup wired to gateway.autoStart config value. Prints status to stderr.
  • 14 unit tests covering all paths: already-running, started, timed-out, spawn-failed.
2026-04-02

Gateway hot reload

  • src/gateway/hotReload.ts — HotReloader class: SIGHUP-driven config reload, channel diff + reconnect, scheduler reinit.
  • ChannelManager gains connectedChannelIds, disconnectChannel(), reconnectChannel().
  • JobScheduler.reinit() — cancels all handles, reloads + reschedules from SQLite.
  • SIGHUP handler in daemon.ts now fires all registered hot-reload handlers.
  • 7 unit tests covering all reload scenarios.
2026-04-01

Gateway daemon & chat channel bridges

  • Gateway daemon — persistent process with PID file, signal handling (SIGTERM/SIGINT/SIGHUP), graceful shutdown, systemd/launchd compatible.
  • WebSocket server (port 7878) + HTTP REST API (port 7879) via Hono — /health, /status, /sessions, /jobs.
  • SQLite persistence via better-sqlite3 (WAL mode) — sessions, jobs, job_runs tables.
  • JWT auth via jose (HS256) — disabled in dev mode.
  • Job scheduler — cron, one-shot, interval jobs persisted to SQLite.
  • Discord bridge — discord.js, mention-gated in guilds, DM always-on.
  • Telegram bridge — grammy, mention-gated in groups, Markdown with plain-text fallback.
  • WhatsApp bridge — Baileys 7.x, QR first-link, persistent auth in ~/.agem/wa-auth/, composing indicator.
  • Matrix bridge — matrix-js-sdk, auto-join invited rooms, typing indicator, history replay guard.
  • TUI auto-connect hook — useGatewayClient probes health on mount, opens WS, reconnects with backoff, falls back silently.
  • Install scripts — install.sh (macOS/Linux/WSL) and install.ps1 (Windows PowerShell).

v0.1.0

v0.1.0 — 2026-03-25

Phase 5

Standalone binary distribution

  • esbuild bundle to dist/agem.js
  • install-bin script copies to ~/bin
  • /selfhost command for structured self-hosting iterations
  • /release [--bump patch|minor|major] — build → test → bundle → git tag pipeline
Phase 4

Frontier capability

  • Subagent delegation — runSubagents() + /delegate for parallel focused sub-tasks
  • Semantic code index — /index command and codeIndex tool with file-cache + content-hash invalidation
  • Telemetry hooks with per-turn and per-session aggregation; /telemetry command
  • Git-native workflow commands — /commit, /branch, /pr, /plan, /review, /release
Phase 3.5

TUI overhaul

  • Token usage tracking and UsageBar with cost estimates and premium quota display
  • Interactive model / mode / reasoning-summary-length selectors in the header
  • Markdown rendering via marked lexer; streaming partial-fence fix
  • Collapsible tool output with E toggle
  • Vim-style input mode (ESC / i / a), persistent input history, tab-completed slash commands
  • commander-based CLI; --message flag for non-interactive scripting / CI
Phase 3

Ecosystem parity

  • MCP client integration (stdio + SSE, multi-server merge)
  • Project-level instruction files (.agem/instructions.md) injected into every turn
  • Context compactor (/compact) with LLM-generated summaries
Phase 2

Workflow quality

  • Session persistence to ~/.agem/sessions/<id>.json; --resume rehydrates store
  • Slash command registry with aliases and tab completion
  • Automated verification loop (/verify) with bounded retries
Phase 1

Reliability foundation

  • Three-tier safety policy: read-only / write / dangerous with execution modes
  • Workspace path guard preventing directory traversal
  • Patch-based editFile replacing whole-file overwrites
  • Shell execution hardening: argument validation, no shell interpolation, path restriction
  • Git status/diff wrappers (/status, /commit)
