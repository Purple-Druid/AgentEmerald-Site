Features

Everything agem can do — from the terminal TUI to chat bridges, daemon, safety system, and extensibility.

🖥️

Terminal UI

A full-featured terminal interface built for developers who live in the shell. Streaming responses, rich markdown rendering, and full keyboard control — no mouse required.

  • Streaming markdown output with syntax highlighting rendered inline
  • Live tool call sidebar — see every tool call as it happens (j/k to navigate, E to expand, f to filter by risk tier)
  • Animated spinner with live status messages during agent processing
  • Diff previews shown before any file write — approve or reject before changes land
  • Approval dialogs for dangerous tool calls in normal execution mode
  • Smooth scrolling through long responses with PgUp/PgDn
  • Press ? to open the keyboard help overlay at any time
  • Context usage bar showing token consumption at a glance
  • Model and execution mode selectors visible in the status bar
  • Vim-style input handling for power users
🤖

Agent Capabilities

agem is a full autonomous coding agent. It reads your code, writes patches, runs commands, researches the web, delegates subtasks, and indexes your codebase for semantic search.

  • 21 built-in tools covering file I/O, shell execution, web research, memory, and more
  • Two model backends: GitHub Copilot and GitHub Models (no separate API key required)
  • Plan mode — reason through a task before executing any writes
  • Web research via DuckDuckGo search + URL fetch
  • Background tasks — spawn long-running jobs and check progress asynchronously
  • Git worktrees — isolated branches for parallel experiments, merged back on exit
  • Subagent delegation — spawn child agents to handle subtasks concurrently
  • Semantic code indexing — codeIndex tool queries symbols, functions, and types
  • Context compaction — automatically summarizes old messages to stay within token limits
  • Provider routing — route different task types to different models automatically
  • Git auto-commit after edits to maintain a clean working tree
  • Verify loop — run build/test commands after edits and iterate until passing
⚙️

The Gateway Daemon

The gateway is a persistent background process that keeps agem alive after you close the terminal. It exposes a WebSocket and HTTP API, manages sessions in SQLite, and runs scheduled jobs.

  • Persistent process — survives terminal closes, SSH disconnects, and reboots
  • WebSocket API on port 7878 for real-time streaming
  • HTTP REST API on port 7879 for programmatic access
  • SQLite-backed session storage — all conversations persisted automatically
  • Job scheduler — cron, interval, and one-shot jobs defined via HTTP or WS
  • SIGHUP hot reload — update config without restarting the daemon
  • Auto-start on agem launch — gateway starts if not already running
  • TUI auto-connect — the terminal UI connects to the gateway automatically
  • Channel adapters (Discord, Telegram, WhatsApp, Matrix) run inside the gateway
📱

Chat Platform Bridges

agem's gateway bridges your codebase to the messaging platforms you already use. Each conversation gets its own isolated session — as if you opened a fresh terminal.

  • Discord — @mention the bot in any channel or DM to start a session
  • Telegram — powered by grammY; responds to any message in allowed chats
  • WhatsApp — QR code first-link, then persistent auth; works from your phone
  • Matrix — auto-join invited rooms; works with self-hosted homeservers
  • Per-conversation session isolation — each chat thread has its own agent session
  • Session key namespacing — sessions are scoped by channel + conversation ID
  • Streaming responses delivered as message edits (where supported)
  • File upload support — send a file and agem reads it into context
🛡️

Safety & Execution Modes

agem gives you granular control over what the agent is allowed to do. Every tool call is classified by risk tier, and you choose how much autonomy to grant.

Mode Description Writes Shell Approval
auto-approve Full autonomy — all tools run without prompting ✅ ✅ Never
normal Default — dangerous tools require approval ✅ 🟡 Dangerous only
read-only No writes — safe for auditing and exploration ❌ ❌ N/A
plan Reason and plan; no execution ❌ ❌ N/A
  • Tool risk tiers: read-only, write, dangerous — shown in the TUI sidebar
  • Workspace path guard — agent cannot read or write outside the configured workspace
  • Diff previews displayed before any writeFile or editFile call
  • Approval dialogs for shellExec, taskCreate, taskStop in normal mode
  • Tool tier filter in sidebar — hide low-risk calls to focus on what matters
💾

Session Management

Sessions are first-class citizens in agem. Every conversation is saved, searchable, and branchable. Pick up where you left off — or fork a session to explore a different approach.

  • Auto-save — every message persisted to SQLite immediately
  • Resume any session with /sessions resume <id>
  • Fork a session with /branch-session — diverge without losing the original
  • Merge sessions back with /sessions merge <id>
  • Diff two sessions with /sessions diff <id> <id>
  • Full-text search with /sessions search <query>
  • Export sessions as Markdown or JSON with /sessions export
  • Auto-naming — agem generates a descriptive name for each session automatically
  • Pin important sessions so they appear at the top of the list
  • Tag sessions for organisation (/sessions tag <id> <tag>)
🧩

MCP & Extensibility

agem implements the Model Context Protocol as a client. Drop any MCP server into your config and its tools become available to the agent automatically.

  • MCP client supporting both stdio and SSE transport modes
  • List connected servers and their tools with /mcp
  • Per-project instructions in .agem/instructions.md — automatically prepended to system prompt
  • Command aliases with /alias <name> <command> — create shortcuts for common prompts
  • MCP servers auto-loaded on gateway start — no manual activation needed
  • Tool namespacing — MCP tools prefixed with server name to avoid conflicts
